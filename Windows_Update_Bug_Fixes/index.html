Windows11 因硬盘格式更新失败修复记录

发表于 2023-08-17 分类于 Tutorials 阅读次数： 本文字数：
起因是我当初换系统盘时选择了错误的格式


在8月中旬我收到了Windows 11 Version 22H2的08积累更新（KB5029263），但我无法成功安装此次更新，每次尝试的最后都会提示“某些操作未按计划进行,不必担心,正在撤销更改”，并恢复到原版本重启。

起先我以为是Windows更新的bug，可能存在驱动的一些不兼容，但经过初步研究，网上关于该问题的讨论比较罕见，主要的解决方法为使用微软官方的win11镜像恢复系统和建立ESP和MSR分区。我按照ESP分区建立方法试图用DiskGenius创建该分区，却发现该选项一直是灰色。于是我又尝试了官方win11镜像，居然提示“抱歉，我们无法确定您的电脑是否可以运行Win11”，可我电脑目前运行的就是win11。

这时我意识到可能是我当初更换系统盘时没用留意过系统盘的格式，可能遗留了问题。于是我得想办法建立MSR分区（Microsoft Reserved Partition）。DiskGenius无法创建该分区，DISKPART工具也报错不是GPT格式无法创建，我才发现我系统盘目前为MBR格式。

最后的解决方法比较简单，且不会清除系统盘上的任何个人数据，分以下几步

  1. 使用PE启动电脑，打开DiskGenius，（备份系统盘），选择硬盘并点击转换分区表类型为GUID格式。选择硬盘下的系统盘并点击建立ESP/MSR分区。为ESP盘分配盘符
  2. 使用UEFI引导修复工具，选择ESP盘和系统所在位置，修复引导
  3. 重启后快速按出启动选项（F10），选择能正常开机的启动项，如果蓝屏就关闭电源重试
  4. 进入系统后下载破解版EasyUEFI软件，删除多余的启动项，我是删除分区开始扇区2048的那项，如果删错就重复步骤23
  5. 重启后电脑应该能正确进入系统并正常更新了

终于搞定了，看来Windows使用中可能遇到的问题还挺多的，官方的说明完全没有提到过该状况，多亏网友的提示才成功解决。
