参加Global Game Jam 2023是怎样的体验

发表于 2023-02-07 分类于 Events 阅读次数： 本文字数： 2.6k
我的第一次game jam（可能也是最后一次）


昨天参加了线下的global game jam 2023。近年的ggj因为疫情持续五天，每个站点选48h举办活动，国内一般在最后两天举办。题目是提前公布的，23年的主题是ROOTS，也就是根。我在开始前构思了大致方案，是一个用鼠标在屏幕上画线模拟根，并在根生长的过程中收集资源的生存游戏。

活动正式开始是2月3日的下午5点。本来想既然参加线下站，直接去线下找队友应该没问题，所在城市的站点在一个游戏公司的写字楼里。到了站点发现大家全都分好组了。。。门口可以领代表职能的牌子，我本来拿的程序，转了一圈又换成全能（最开始没想到会要solo）。后来有落单人在群里发了组队信息，我回应了。那位大哥说自己想做策划，没有开发经验，也是第一次参加。这时候明智的做法就是拒绝他，不过我没想这么多，把主题和策划告诉了他。活动正式开始后赞助方（几个小的游戏公司）介绍了自己的项目（为免费的零食饮料点赞），然后播放了ggj统一的开幕视频。

接下来我和大哥对游戏的主题进行了讨论。他认为我的方案大体可行，不断在想改进意见，我以为这是良好的方向（毕竟他担任策划）。不过他给的意见大多都泛善可陈，没法通过我的质疑。到了晚上六点，我们还没有取得实质的进展，他直接回家了，而我留下来开发。我会unity和construct 2两个引擎，因为团队只有两个人，想着还是c2合适一点。晚上我在会场搭建了核心玩法，创建树根分叉的功能遇到了bug，卡了我很久，其实是我复用之前的事件时忘了改参数。

回到家继续搭建其他功能，凌晨时终于实现了全部画线功能。突然失联6小时的队友发信息给我，说我的方案想不出新的点，要退出自己单干。此时我无话可说，先不论我有没有能力独自完成游戏，一个啥都不会的新手，真的有可能在ggj学会做游戏吗？结局不出我所料，接下来他就没有再出现了。

于是我就被坑着开始了我的solo之旅。之前的开发计划打乱了，单人开发难度不小，既要兼顾游戏的概念和玩法设计又要兼顾具体的实现。第二天凌晨我研究了不少关于根的话题，结果在翻转电台里找到了我愿意表达的内容：根基性。我想把根基性作为游戏的暗线，始终构思不出恰当的方案。起初想呈现人类文明的根基性，也就是电台里讲的自然法、仁、中庸、交往理性、努斯、明智、认识你自己、天…抽象的哲学概念要落实在玩法上并不容易。我尝试摘录一些句子来传达这些思想，但却找不到合适的内容。后来想或许传达个人的根基性，也就是人的故乡、家庭之类的内容会实际一些。想到这我就睡觉了，差不多2点睡到了中午12点。

由于失去了团队，第二天我选择在家里开发。起来后吃了中饭，我先做了随机生成水和矿物质并获取资源的程序。接下来打算用ai跑一点家族成员的图片当游戏素材。我之前注册了scenario，是可以免费ai生成美术资源的网站，但由于对ai绘画操作的熟练度不够，没有准备素材来做自己的模型，产出效率比较低。经过艰辛的尝试，我终于得到了一套比较满意的肖像，并将其转成黑白色调后导入游戏。实现了游戏中创建分叉后会在分歧点上生成一副肖像。第二天晚上我一直在思考应该怎样编写游戏的文字内容。本来我依旧想配上摘录的哲学思想，不过在读到维特根斯坦的内容后突发奇想可以介绍一下维特根斯坦之家，这是奥地利最富有的传奇家族之一。同时，在读了海德格尔的一篇演讲后，“我们像植物一样，不管我们承认与否，为了能够在天空中开花结果，必须保留从土地上生长的根基，方能在天穹中开花结果”一句深深吸引了我，这不正好是我想表达的主题吗？光思索游戏的文字内容就从第二天晚上想到了第三天凌晨4点，之后睡到了11点，剩下的时间来不及做什么大改动了。

第三天起来便赶去活动现场，连中饭都没来得及吃。我简单叙述了维特根斯坦的生平，并以L作为称呼。游戏中随着根的生长，会不断出现介绍他早年经历的字幕，而创建分叉后会显示我改编过的他的家族成员介绍和肖像，虽然感觉这个机制有些蹩脚，但我实在想不出别的创意了。剩下的时间我给地图加了迷雾和探照灯，用鼠标指向资源所在的方向才能看到附近的资源，本来还想做害虫但觉得和游戏的思想传达不符。大概下午3点我结束了开发，试着通关了一遍，对难度进行了调整。随后我开始进行游戏的打包和上传工作，赛事规定还要求我制作一段介绍游戏的视频。5点活动结束，我也完成了全部工作，去吃了晚饭。

晚上是各组的展示和宣讲环节。算上我一共有三组solo，在solo组里感觉我的完成度算不错。别的组有从业者，也有大学生队伍，不同组的对比明显。学生队伍比较切活动的主题，好几组和我的玩法类似，都是模拟树根的生长。从业者就会出一些有创意的玩法，我印象比较深的一组是通过甩鼠标建立一根不断吸收周围元素的根，在三种互克的元素间切换，遇到克制元素就会爆炸。游戏的视觉和音效都非常惊艳，是很酷也很成熟的设计。还有一组控制太极两端的双人游戏，不断拉扯的同时吸收周围的元素得分，完成度也很高，然而，它似乎与活动的主题不是很相关（除了太极两端之间的连接线）。我也进行了展示，虽然因为社恐有点语无伦次，不过主持人说我的作品很棒，即便是恭维也觉得很开心。

当天是元宵节，市里举行了烟火晚会，因此回家的路上人很多。我到家躺下很快就睡着了。

写了这么多，最后总结下。

  • 不要幻想在现场能够找到优秀的队友。我认为一个1策1程1美的团队比较适合我，其中策划负责统筹项目和设计游戏内容，程序员和美术师在遇到困难或者需要帮忙时可以互相协作，例如编写代码模块或者使用AI绘图。团队必须由三个有开发经验的人组成，千万不要让一个什么都不会的策划加入其中。我希望下次能找到一个完整的团队参加，不想再一个人开发了。
  • 要学的还有很多。虽然solo时用construct2并没有遇到太大的障碍，但对主流开发引擎unity的掌握显然不够。游戏策划上也有太多要学，对玩法的思考太少。人文社科的积累对游戏开发真的有帮助，但是落实到和具体玩法结合上需要不断的思考。 经验的缺乏是我最大的障碍，需要未来多玩多看多总结。
  • ai可以极大的辅助开发。这次没有chatgpt和scenario的帮助能不能完成游戏真的很难说。
  • 没必要过度准备，在短暂的时间里挑战自我别有一番乐趣，极限下的努力和挣扎更值得怀念。
  • ggj的作品相比堆积美术和关卡素材，更应该关注创意，关注玩法，关注自我表达。我会希望未来做的游戏在上面三块中起码有一块沾边，唯有探索的过程才能带来成就。
